COVID-19 in NYC: Charted

Project Link | GitHub Repository

A website dedicated to charting the COVID-19 pandemic, more specifically in New York City. The website fetches and charts data without a backend. Built using JavaScript.

The Inspiration: With the COVID-19 pandemic well underway, the NYC Department of Health began releasing pertinent data on GitHub in late March. Realizing the potential to tell a story using the data, I created COVIDinNYC.com.

The Challenge: Rather than simply charting the data (which the NYC Department of Health had already done), I wanted to compare the data given by the Department of Health with other data sets.

Challenges

  1. No Backend Wanted
    • Because of the way the Department of Health decided to release the data (a .csv on GitHub), I decided to use JavaScript to fetch the data and chart them.
      • This allowed me to avoid hosting the website on a virtual machine, instead hosting it on GitHub Pages for free.
      • This also meant that should the Department of Health decide to change the model of the data in the .csv's, my website would break.
  2. Interactive Charts
    • What's awesome about front-end development is allowing visitors to interact with data. This manipulation of data allows users to better understand the data they're seeing, and I saw it as a crucial feature for the website. Keeping that in mind, I began searching for JS charting libraries, and went with chart.js
  3. Adding/Subtracting Dates
    • A necessary requirement in the making of the "Pueyo Estimate" model was allowing users to select different variables to see different projections. One of these variables was dates. I soon became very aware of the headaches associated with adding/subtracting dates and time.
      • To solve this issue, I implemented moment.js
